計算の詳細

「いまの空色」が空の色を求める計算の、数式レベルの解説です。概要は空の豆知識をご覧ください。すべての計算は端末内で完結し、外部データは使いません。

1. 太陽高度

天文年鑑（Astronomical Almanac）の簡易太陽位置式を使っています。J2000.0 元期（2000年1月1日12時UT）からの経過日数を d とすると:

g = 357.529 + 0.98560028 d　（平均近点角）
q = 280.459 + 0.98564736 d　（平均黄経）
L = q + 1.915 sin g + 0.020 sin 2g　（視黄経）
ε = 23.439 − 0.00000036 d　（黄道傾斜角）

視黄経 L と黄道傾斜角 ε から太陽の赤経 α・赤緯 δ を求め、グリニッジ恒星時 GMST と経度から時角 H を出します:

α = atan2(cos ε · sin L, cos L)
δ = asin(sin ε · sin L)
GMST = 280.46061837 + 360.98564736629 d
H = GMST + 経度 − α

最後に、緯度 φ と合わせて高度 h が決まります:

sin h = sin φ · sin δ + cos φ · cos δ · cos H

この簡易式の精度は太陽位置で 0.01° 程度。本アプリの用途（色とフェーズの決定）には十分です。

2. 大気差（見かけの高度への補正）

地平線付近では大気の屈折で太陽が浮き上がって見えるため、ベネットの式（Bennett, 1982）で幾何学的高度を見かけの高度に補正します。R の単位は分角です:

R = 1 / tan( h + 7.31 / (h + 4.4) )
h見かけ = h + R / 60

地平線での補正はおよそ 0.5°（太陽の視直径ぶん）。h < −1° では補正を打ち切ります。

3. 照度（lux）モデル

太陽高度と快晴時の地上水平面照度の関係を、2つの領域に分けて近似しています。

太陽が高いとき（h ≥ 3°）:

E = 133800 × sin(h)1.15　[lux]

薄明のとき（−18° < h < 3°）は、log10E を高度に対する折れ線で補間します:

h = 3°log₁₀E = 3.65約 4,500 lux
0°2.60約 400 lux
−6°0.53約 3.4 lux（市民薄明の終わり）
−12°−2.10約 0.008 lux（航海薄明の終わり）
−18°−3.22約 0.0006 lux（星明かりの夜空）

照度は真昼から星空まで8桁変化するため、画面の明度 B は log₁₀E を最小〜最大で 0〜1 に正規化した値を使います。人間の明るさ感覚が対数的であることとも整合します。

4. 色の合成

色は「色相」と「明るさ」を分けて扱います。太陽高度ごとに画面上端・下端の基準色（0〜1 の RGB）を定義したテーブルを持ち、間の高度は線形補間します:

−18°夜
−12°天文薄明
−8°航海薄明
−4°市民薄明
0°日の出/日の入り
3°ゴールデンアワー
10°朝夕
30°昼
60°真昼

見本は基準色をそのまま表示したもの（明度 B = 1 相当）。実際の画面では明度が掛かります。

最終的な表示色は、基準色 C に明度 B を掛け、夜でも真っ黒にならないよう最小の環境光（RGB でおよそ 5, 7, 18）を足したものです:

表示色 = 環境光 + C × B × 255

フェーズ名（「市民薄明」など）は、現在の高度に最も近い側のテーブル行から決まります。

5. ゴールデンアワーなどの時刻の求め方

「この場所の今日のゴールデンアワーは何時か」は、逆算ではなく走査で求めています。その日の0時から24時まで1分刻みで高度を計算し、フェーズの境界（高度がしきい値をまたぐ時刻)を検出する方式です。単純ですが、極地の白夜・極夜のような「境界が存在しない日」も自然に扱えます。

6. タイムゾーン

都市ごとの時刻は IANA タイムゾーンデータベース（Asia/Tokyo など）に基づいて現地時間と UTC を相互変換しています。夏時間も自動で反映されます。太陽位置の計算自体は常に UTC で行うため、時差の影響を受けません。

このモデルが考慮していないもの

雲・エアロゾル（黄砂や煙霧）・光害・標高・地形による遮蔽・月明かりは考慮していません。表示されるのは「快晴・海面高度・遮るもののない地平線」を仮定した理論値です。だからこそ、通信なしで世界中のどの場所・どの時刻の空も描くことができます。
